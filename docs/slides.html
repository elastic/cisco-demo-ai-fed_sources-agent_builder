Search · future

Serverless Search & federated blob sources

Enterprise Search roadmap S3 · GCS · Azure One Serverless project

Today’s lab uses managed indices + connectors in one Serverless Search project. The roadmap extends that same project to federated object storage — query archives in place without sizing a second cluster on cold data.

Register inexpensive blob prefixes as federated data sources; keep hot corpus on managed Search; one ES|QL story across both.

Why it matters at Cisco: years of JSON/Parquet exports, support archives, and compliance buckets become searchable without re-copying everything into a sized frozen tier.

Architecture (future)
